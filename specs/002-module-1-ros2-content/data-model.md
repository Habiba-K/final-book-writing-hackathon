# Data Model: Physical AI & Humanoid Robotics Textbook

**Date**: 2025-12-06

## Key Entities

### 1. Module

*   **Description**: A top-level organizational unit for related chapters, representing a major section of the textbook (e.g., "ROS 2 Fundamentals", "Digital Twin").
*   **Attributes**:
    *   `id`: Unique identifier (e.g., "module-1", "module-2").
    *   `title`: Display name (e.g., "Module 1 â€“ ROS 2 Fundamentals").
    *   `prerequisites`: Text describing required prior knowledge or software.
    *   `learning_outcomes`: Text describing what students will achieve.
    *   `architecture_overview`: Text/diagram reference for module's architectural components.
    *   `code_failure_modes`: Common errors and troubleshooting.
    *   `safety_rules`: Critical warnings.
*   **Relationships**: Contains many Chapters.

### 2. Chapter

*   **Description**: A learning unit within a module, containing detailed content, code examples, and exercises.
*   **Attributes**:
    *   `id`: Unique identifier (e.g., "chapter-1-gazebo-setup").
    *   `title`: Display name (e.g., "Gazebo Setup").
    *   `markdown_content`: The main body of the chapter, including text, images, and embedded code examples.
    *   `learning_objectives`: Bulleted list of specific, measurable outcomes.
    *   `worked_examples`: References to or inline descriptions of key demonstrations.
    *   `numbered_instructions`: Sequential, testable steps.
    *   `final_exercise`: Hands-on task to test comprehension.
    *   `language_tags`: Array of languages used in code examples (e.g., "Python", "bash", "XML", "YAML").
*   **Relationships**: Belongs to one Module; contains many Code Examples.

### 3. Code Example

*   **Description**: A runnable code snippet demonstrating a specific concept or instruction.
*   **Attributes**:
    *   `content`: The actual code block.
    *   `language`: Programming or scripting language (e.g., "python", "bash", "xml", "yaml").
    *   `annotations`: Inline comments or preceding text explaining key lines.
    *   `execution_commands`: Optional terminal commands to run the example.
*   **Validation Rules**: Must be runnable and produce expected output in a configured environment.
*   **Relationships**: Belongs to one Chapter.

### 4. URDF Model (Specific to relevant chapters)

*   **Description**: Robot model definition using XML, describing the kinematic and dynamic properties of a robot.
*   **Attributes**:
    *   `xml_content`: The complete URDF XML structure.
    *   `links`: Definitions of robot body parts (visual, collision, inertial elements).
    *   `joints`: Definitions of connections between links (types like revolute, prismatic, fixed; axis definitions).
    *   `kinematic_tree`: Implicit structure defined by links and joints.
*   **Validation Rules**: Must be valid XML and parse without errors in ROS 2 tools.
*   **Relationships**: Belongs to one Chapter (e.g., Chapter on URDF).

### 5. Other Implicit Entities (Content Structure)

*   **Docusaurus Site**: The overall static website generated by Docusaurus.
*   **Sidebar**: Navigation component listing modules and chapters.
*   **Search Index**: Data structure enabling full-text search across all content.
